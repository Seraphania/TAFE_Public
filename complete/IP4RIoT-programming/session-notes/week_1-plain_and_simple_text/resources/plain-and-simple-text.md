## Lesson Overview: Session 1 - "Plain and Simple Text"
*Objective:*  
Introduce new programmers to the concept of plain text, its importance in programming, the basics of markup languages and markdown, and familiarization with programming editors.

### What is Plain Text?
**Plain text** is unformatted text, visible as simple characters. WYSIWYG (What You See is What You ~~Get~~ Got). The raw content of the file is visible and it is assumed that every *byte* in the file, represents a character (more or less).  
Given this, it is very limited in that it can't do fun things like FORMATTING . It is just text. Plain text.

### What isn't plain text?
* As you may have noticed, the world of computers consists of a bit more than just plain text: videos, images,  presentations, rich documents, web pages ... web pages. 
* Generally speaking, there are two ways to get RICH (text): 
  * **Binary** - it is all numbers in the end. It is up to our programs to interpret this: videos, images, and presentations mostly use this approach as it is very compact
  * **Markup** - these are still just text documents but the text is written in such a way that it makes interpreting that text easier for a program (e.g. a web browser) such that we can add rich elements like all the formatting you see here (which ultimately is just HTML, which is just plain text with lots of \<tags>).

### What do programmers use?
Programmers use plain text for coding because it's universally readable by both humans and computers. It ensures compatibility across different platforms and environments, making it essential for writing and sharing code.

### Markup Languages and Markdown
Markup languages like HTML and XML structure data in plain text. Markdown is a lightweight markup language with plain text formatting syntax designed to be converted to HTML. It simplifies the process of writing content on the web.

### Programming Editors
These are specialized text editors optimized for writing and editing code. They offer features like syntax highlighting, line numbering, and code formatting to make coding more efficient.

### Activities:
#### Activity 1: Think/Pair/Share - Word Document vs. Plain Text Document
* **Objective:** Understand the difference and relative advantages/disadvantages between Word Documents (rich text) and Plain Text documents.
* **Process:** Individually think about the differences, then pair up to discuss and share findings with the class.

#### Activity 2: Think/Pair/Share - Why Do Programmers Use Text Files
* **Objective:** Explore reasons behind the preference for text files in programming.
* **Process:** Reflect individually, discuss in pairs, and share insights on why text files are fundamental for programming tasks.

#### Activity 3: Practical Task - Create a Markdown Document
* **Objective:** Apply knowledge of plain text and markdown by creating a markdown document that replicates a formatted Microsoft Word document.
* **Process:** Use an Integrated Development Environment (IDE) to create a markdown (.md) file, attempting to match the formatting of a given Word document example.

This session lays the foundation for understanding the basic tools and formats used in programming, preparing participants for more advanced topics in software development.

## Markdown cheatsheet
* **Headers:** Use # for headings. More #s mean smaller headings: # Header 1, ## Header 2...
* **Bold:** Wrap text with **, like \*\*bold\*\*: **bold**.
* **Italic:** Use *, like \*italic\*: *italic*.
* **Links:** \[Text](URL), like \[Google](http://google.com): [Google](http://google.com).
* **Images:** \[Alt text](URL):  ![Alt text](URL), similar to links but starts with !.
* **Lists:** Use - or * for unordered lists, 1., 2., for ordered.
* **Blockquotes:** Start a line with >.
* **Code:** Use backticks ` for inline code, ``` for blocks.
