# Creating Tables 
## Portfolio Exercise

### Queries 
For each of the following scenarios, construct a SQL query that will create the table.  

You need to pay particular attention to the data types you choose for the table and pick the most appropriate field type for the data you want to store. On one hand, you need to allow for edge cases, such as incredibly long surnames. On the other hand, if there are no edge cases, you need to store only as much information as you need. For example, if you wanted to store a human’s age, using anything bigger than an unsigned tiny int (which can store a number from 0-255) would be a waste of resources. There are extreme edge cases, but they only go so far, and from 0-255 is more than enough for any edge case we will ever encounter.  

Again: You should think very carefully about data types. If an answer is otherwise correct but your choice of data types is unnecessarily wasting server resources or not accounting for edge cases that genuinely exist even if they’re rare, then it will not be considered satisfactory.  

In order to determine the best datatype for a given situation, some research may be required.  

For each answer, you should also run the query in SQL just to make sure the syntax is correct.  

1. Create a table to store the current stock prices of companies. So, the name of the company, the stock ticker and the current share price.  
2. Create a table to store information about a car. So: make, model, number plate and year.  
3. Create a table to store basic customer details at a shop. So: Name, surname, gender, date of birth, phone number and address.  
4. Create a table to store information about a baby born at a hospital. So: Name, surname, gender, weight, height, and when they were born.  
5. Create a table that stores various famous landmarks and their distance from Perth. The table should store the name of the landmark, a star rating out of five, the longitude and latitude, the city or town and the country. 

When you’re done, show the lecturer your tables. This is not to mark them, as such, just just to check you got them up and running in SQL okay.
